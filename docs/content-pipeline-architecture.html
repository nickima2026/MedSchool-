Three-Source Content Pipeline

XIB.io Lifelong Learning Platform Architecture
April 11, 2026 — Selway Solutions LLC

Design Principle

Every medical school department, every hospital system, every CME provider delivers lectures through one of three channels: YouTube recordings, YouTube Live streams, or Zoom sessions. If XIB.io handles all three, a graduate's learning platform never goes stale. The student's bespoke instance persists from enrollment through their entire career — AI as lifelong mentor.

Three Content Sources

Live — Deployed

Source 1: YouTube Recordings

Student pastes any YouTube URL. System extracts transcript via innertube API (3-client fallback strategy). Claude generates structured study notes.

Trigger: Manual — student pastes URL

Auth: None required

Example: Student finds a cardiology lecture, pastes URL, gets USMLE-style study notes in seconds.
New — Building Now

Source 2: YouTube Live Series

Scheduled auto-capture for recurring lecture series. Monitors channel RSS feed, detects new uploads, auto-runs transcript + AI pipeline.

Trigger: Scheduled task (weekly cron)

Auth: None — public RSS feed

Example: Primary Children's Pediatric Grand Rounds auto-captured every Thursday. Notes appear in student's Pediatrics course.
Future — Design Phase

Source 3: Zoom Sessions

Captures transcripts from authenticated Zoom sessions — department Grand Rounds, CME lectures, clinical conferences.

Trigger: Manual or email-triggered

Auth: Zoom OAuth required

Example: Stanford Anesthesia Grand Rounds. Student authenticates once, transcripts auto-pulled from cloud recordings.

Implementation Roadmap

Source 1: YouTube Recordings (Complete)

Manual paste → transcript → AI analysis. Deployed and working on MSIB.io.

Source 2: YouTube Live Auto-Capture (In Progress)

channel-monitor.js and auto-capture.js built. Next: deploy to Vercel, create Cowork scheduled task, test with PGR PCH channel, add subscription UI to dashboard.

Source 3: Zoom Sessions (Design Phase)

Zoom OAuth integration, cloud recording transcript API, email-trigger detection for session announcements. Target: Stanford Anesthesia Grand Rounds.

Storage Migration: localStorage → Supabase

Move from browser localStorage to Supabase for persistent, cross-device access. Enables the mobile app and multi-device workflows.

Offline Mobile App (Swift/Xcode)

iOS app with locally cached text content. Study, review, and self-test without connectivity. Syncs when online.

Cross-Platform Rollout

Port content pipeline to LSIB.io (law), VSIB.io (veterinary), and all XIB.io platforms. Same three-source architecture, discipline-specific AI analysis prompts.
